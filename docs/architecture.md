# 架构与兼容策略

## 系统边界

```text
浏览器
  │ HTTPS / HTTP
  ▼
kdae-panel
  ├── 认证与会话：SQLite + Argon2id
  ├── 配置事务：临时文件、validate、备份、原子替换、回滚
  ├── dae 适配：固定参数调用公开 CLI
  ├── 服务管理：systemctl
  └── 日志读取：journalctl JSON
        │
        ▼
      dae.service
```

面板和 dae 是独立进程。面板不链接 dae 的内部包，不要求修改 dae，也不需要向上游增加 Unix Socket API。

## 稳定契约

面板只依赖以下外部契约：

| 契约 | 用途 | 变化风险 |
|---|---|---|
| `dae --version` | 显示安装版本 | 低 |
| `dae --help` | 基于能力而非版本号启用功能 | 低 |
| `dae export outline` | 动态配置结构 | 低 |
| `dae validate -c` | 使用当前二进制校验候选配置 | 低 |
| `dae reload` | 应用配置 | 低 |
| `.dae` 入口配置 | 配置的唯一真实来源 | 中 |
| systemd 单元 | 生命周期与资源状态 | 低 |
| journald JSON | 近期日志 | 低 |

不稳定的内部对象、eBPF Map、Go 包、内存布局和普通日志文本均不作为控制契约。

## 配置事务

保存操作在进程级互斥锁内串行执行：

```text
检查客户端提交的旧 SHA-256
  → 在入口配置同目录写入候选 .dae 文件
  → 继承入口文件权限
  → fsync 候选文件
  → dae validate -c 候选文件
  → 将旧配置写入 0600 备份文件
  → 同文件系统原子替换入口文件
  → dae reload
  → 失败时恢复旧磁盘配置
```

候选文件与入口配置位于同一目录，确保 dae 的相对 `include` 路径仍按入口目录解析。SHA-256 乐观锁用于阻止两个浏览器或外部编辑器互相覆盖。

## 动态结构

`dae export outline` 返回版本、叶子类型、字段映射、数组标记、默认值、必填标记和字段说明。面板直接展示当前二进制返回的结构，不维护固定字段清单。

原始 `.dae` 文本仍是权威数据。动态结构用于发现和浏览能力，而不是将未知文本解析后重新生成，因此上游新增语法不会被旧面板删除。

## 当前观测边界

在不修改 dae 的前提下，面板可以可靠获得：

- systemd 运行状态、PID、内存、累计 CPU、任务数和重启数；
- journald 日志；
- dae 版本、命令能力和配置结构；
- sysdump 诊断信息。

以下数据没有稳定公开接口，当前不伪造或解析内部实现：

- 实时连接列表；
- 每条路由的命中统计；
- dae 维度的精确流量；
- 内部 DNS 缓存；
- 每个出站节点的真实运行时健康状态。

未来若 dae 提供稳定 API，这些能力可以作为可选适配器增加，不影响现有 CLI 路径。
